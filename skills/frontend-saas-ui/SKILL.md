# Frontend SaaS UI

## Description

Use this skill to design and implement public-facing Smart Career Hub pages that feel premium, modern, and intentionally crafted rather than template-based.

## When to use this skill

Use this skill when building:
- Landing page
- Login page
- Signup page
- Public SaaS pages
- Marketing sections
- Reusable public layout components such as navbars and footers

## Strict implementation rules

- Build premium SaaS UI with strong hierarchy, clean spacing, and trustworthy visual tone.
- Start mobile-first and scale upward without patching desktop layouts back down later.
- Use shadcn/ui primitives, but customize the composition and styling so the result does not look stock.
- Use Lucide icons for clean, consistent iconography.
- Use Framer Motion for subtle section reveals and lightweight transitions.
- Do not copy generic landing-page templates or overused startup hero patterns.
- Keep sections focused, readable, and conversion-oriented.
- Use gradients only when they improve depth or emphasis; never as decoration alone.

Landing page must include:
- Navbar
- Hero
- Trust indicators
- Problem section
- Solution section
- Features
- How it works
- Product preview
- Pricing placeholder
- Final CTA
- Footer

## Acceptance checklist

- The page feels premium, modern, and SaaS-ready.
- The layout reads clearly on `320px` mobile widths.
- The UI does not resemble a default template.
- Navbar, hero, trust indicators, core sections, CTA, and footer are all present.
- Motion is subtle, smooth, and accessible.
- Buttons, spacing, and typography feel polished and intentional.
